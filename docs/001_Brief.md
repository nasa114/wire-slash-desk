# 企画書（ペライチ）: パーソナルRSSリーダー + AIキュレーション

| 項目 | 内容 |
|---|---|
| 目的 | 海外の文献・ニュース・リリース情報を定期収集し、生成AIでトピック抽出・日本語キュレーションして**個人で**消費する |
| 利用者 | 本人のみ（商用・公開サービスではない） |
| 技術スタック | Node.js / PostgreSQL（pgvector拡張前提）/ MCPサーバー / Dev Container |
| 実装体制 | Claude Code（内蔵 `/goal` による自走運用）+ Codex（GPT-5系）による agentic coding |

## 解決したい課題

- 海外情報源の巡回・言語の壁による情報収集コストが高い
- 「今日は何が話題か」の集約を手作業でやっている
- ニュースアプリ等では自分の関心領域（技術・セキュリティ・リリース情報）に最適化できない

## スコープ

**やること**
1. RSS/Atomフィードの定期収集 — **保存するのはタイトル・URL・公開日時のみ**
2. フィード管理（収集間隔 / 翻訳要否 / **本文取得可否フラグ** / 有効無効）
3. MCPサーバー経由でフロンティアモデル（Claude / ChatGPT等）にデータを公開し、AI側でトピック抽出・日本語要約・キュレーションを実行
4. MCP認証: Bearerトークン（初期）→ OAuth 2.1（本対応）

**やらないこと**
- 本文の一括自動収集（フィード規約・著作権リスクのため。取得はソース単位の許可制＋オンデマンドのみ）
- 多ユーザー対応・再配信・SEO
- 独自のAI推論基盤の構築（推論は外部API / MCPクライアント側）

## 方針・制約

- **コスト最小**: VPS / Render / Supabase / GitHub Actions schedule 等の低コスト構成
- **規約遵守が最優先**: 本文取得はソースごとに手動判断したフラグでのみ許可（例: GitHub Releases系は可、日経のような媒体は不可）
- **DB抽象化 + TDD**: リポジトリパターンでDBエンジン差し替え可能に。MySQL前提の既存コードからPostgreSQLへ移行
- **開発環境**: Dev Container。Claude / Codex のホームディレクトリを永続化し、バックアップスクリプトを整備
- **引き継ぎ事項（過去の検討より）**: 国内データ主権の観点で要約系は「さくらのAI Engine」を採用した経緯あり。脆弱性系/非脆弱性系の2トラックスコアリング構想あり → 本文書群では枠のみ確保し、詳細は別途判断

## マイルストーン

| M | 内容 |
|---|---|
| M0 | 開発基盤: Dev ContainerのPostgreSQL化、エージェント環境の永続化+バックアップ |
| M1 | スキーマ・マイグレーション・リポジトリ層（TDD） |
| M2 | フィード収集（タイトルのみ）+ 収集トリガー |
| M3 | MCPサーバー（読み取りツール群 + Bearer認証） |
| M4 | 設定UI（最小限）/ OAuth 2.1 |
| M5（任意） | pgvectorによる類似検索・トピッククラスタリング、2トラックスコアリング |
